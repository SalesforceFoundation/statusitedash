Batch Data Import

To support Batch Data Import, you must perform some setup tasks. These steps are required for all orgs using the NPSP Data Importer (regardless of whether you plan to use the new Batch Data Import features.)

  • In the profiles of all users who perform data imports using the NPSP Data Importer:
    • Set the NPSP Data Import Batch tab setting to Default On.
    • Set object permissions for the NPSP Data Import Batch object to Read, Create, Edit.
    • In the NPSP Data Import object settings, set field permissions for the NPSP Data Import Batch field to Read Access and Edit Access.
    • Override the Clone, Edit, and New buttons on the NPSP Data Import Batch object to use the BDI_BatchOverride Visualforce page.
  • In the NPSP Data Import Batch page layout:
    • Add all available fields to the layout.
    • Add the Process Batch button to the layout.
    • To schedule the new NPSP 09 - Data Import Batch Processing job, a System Administrator in your org needs to visit the NPSP Settings page. Visiting this page automatically schedules all NPSP jobs. If you enabled the NPSP Setting Don't Auto Schedule Default NPSP Jobs, you'll need to manually schedule the new job in Setup.
  • You may notice some minor layout updates on many pages in NPSP. That is because we updated the underlying Salesforce Lightning Design System version.
  • When the Amount on an Opportunity with an associated Campaign or Recurring Donation is updated from null to a value, Allocations are now created based on the Campaign's General Accounting Unit (GAU) or the Recurring Donation's GAU, instead of the Default Allocation GAU. If the Opportunity is associated with both a Campaign and a Recurring Donation (that have GAUs), the Recurring Donation GAU takes precedence.
  • When Enable Automatic Verification (in NPSP Settings under People > Addresses > Verification API Account Settings) is selected, NPSP no longer tries to verify Addresses that already have the Verified checkbox selected.
  • When verification is attempted, NPSP will now always select an Address's Verified checkbox, regardless of the outcome of the verification. You can use the Verification Status field on an Address to determine if a match was found.
  • #1864: If Audit Fields are enabled through Permission Set, NPSP will insert those fields for new records based on an NPSP Data Import custom field
  • #2636: Campaign Allocations not used if a null Amount field is populated after the Opp was previously saved.
  • #2713: With address verification, Verification Status field not updating correctly
